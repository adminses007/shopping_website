# Code Style and Conventions

## Python Code Style
- Follow PEP 8 guidelines
- Use meaningful variable and function names
- Add docstrings for functions and classes
- Use type hints where appropriate

## Flask Conventions
- Route functions should be descriptive
- Use proper HTTP methods (GET, POST)
- Handle errors gracefully with try-catch blocks
- Use Flask-Login for authentication

## HTML/CSS/JavaScript
- Use Bootstrap 5 for responsive design
- Follow semantic HTML structure
- Use consistent CSS class naming
- JavaScript functions should be well-documented

## Database Conventions
- Use SQLAlchemy ORM
- Table names should be lowercase
- Column names should be descriptive
- Use proper foreign key relationships

## File Organization
- Templates in `templates/` directory
- Static files in `static/` directory
- Configuration in separate files
- Database models in main app file

## Naming Conventions
- Python files: snake_case
- HTML files: snake_case
- CSS classes: kebab-case
- JavaScript functions: camelCase
- Database tables: lowercase
